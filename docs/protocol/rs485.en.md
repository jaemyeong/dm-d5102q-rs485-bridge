## RX Capture

RX bytes are collected with timestamps and exposed to the Web Console as raw/HEX dumps. Overflow metrics are separated into UART overflow, queue overflow, and dropped frames.

## TX Path

1. The user enters a HEX string.
2. Spaces are allowed, but the input is validated as two-digit HEX bytes.
3. The validated byte array is placed on the TX queue.
4. The RS485 driver switches DE/RE into transmit mode.
5. After UART write and flush, the driver returns to receive mode.
